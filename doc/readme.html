What are the Woodstock Components?

The Woodstock components are a set of general purpose UI components that follow Sun Web Application Guidelines (to be supplied), and can be used for developing web applications and portlets at Sun. The components are built on the JavaServer(TM) Faces library, a JSR standard technology for building web components with Java.  The components will work in any web container that complies with Servlets 2.5, JSP 2.1, and JSF 1.2.  Currently, only GlassFish (Application Server 9) meets all these requirements.   See Software Requirements.

The Woodstock components are available in the Visual Web Pack plugin module for Netbeans 5.5, but can also be used standalone to build web applications.  This readme describes how to build and use the standalone Woodstock components.

Related Documentation

  • Component Release Notes are included in the components jar (Woodstock.4.0.1-yyyymmddhhmm.jar).
  • Javadocs for the components are created when you build the components.  An online version will be available in the future.

  • TLD (tag library descriptor) documentation describes how to use the components, is included in the components jar.

Software Requirements

JSF 1.2 / Glassfish v1 or App Server PE 9 - The Woodstock components are based on JSF 1.2, which currently requires you to deploy using Glassfish (Sun Java System Application Server PE 9), the only freely available web container that supports JSF 1.2.  The easiest way to get everything you need to develop apps using the Woodstock components is to get the latest Netbeans 5.5 bundled with App Server 9.  Currently Netbeans 5.5 can be downloaded with AppServer 9 from http://www.netbeans.info/downloads/index.php.You can also download GlassFish from http://glassfish.dev.java.net/public/downloadsindex.html.

JDK 1.5.0 -The Java version should be at least 1.5.0_06, with 1.5.0_08 recommended.   If you will be using the Visual Web Pack, 1.5.0_08 is required.

Ant 1.6.5 - This Ant version is included in NetBeans 5.5. If not using NetBeans, you can get Ant 1.6.5 from the Apache Ant site.

Netbeans  5.5 - Netbeans is not required to use the components, but is recommended because of its source level debugging, and full featured editor for building web applications.  If you use Netbeans, it should be at least version 5.5.  You can download various different bundles that contain Netbeans 5.5 from netbeans.org.

Getting Started

You should first get a good background in JavaServer Faces. Here are a few suggestions:
  • Java EE documentation: http://java.sun.com/javaee/javaserverfaces/index.jsp

  • Tutorial: http://java.sun.com/javaee/5/docs/tutorial/doc/index.html

  • JavaServer Faces, Hans Bergsten, O'Reilly Media, Inc., ISBN: 0596005393

  • Core JavaServer Faces, Geary and Horstmann, Prentice Hall PTR / Sun Microsystems Press, ISBN: 0131463055

Once you know JSF, you need to decide how you want to build your application. There are a few ways to do this, but this document only discusses Netbeans. You can also use a text editor if you code manually.  You can also use the Visual Web Pack for Netbeans.

Netbeans 5.5 is recommended because of its source level debugging, and full featured editor for building web applications.

Build the Source

The Project Woodstock source code distribution includes the code for the components and an example application that demonstrates how to use the components API.  See Software Requirements for dependencies needed to build the source.

Before you can download the source code, you will first need to:
  • register for an account on java.net
  • install CVS on your machine
To download the source code:
  • checkout the source
To build from the command line, presuming you checked out the code into <myWorkspace>:
  • ant -f <myWorkspace>/woodstock/master/build.xml
Alternatively, the Woodstock code comes complete with Netbeans 5.5 compatible project files such that each project can be loaded and build seperately within the Netbeans IDE:
  • Launch netbeans. This task is different for each OS, so consult the Netbeans documentation.
  • From the File Menu, choose: Open Project
  • In the Open Project dialog, navigate to <myWorkspace>/woodstock
  • Select one of the available projects: example, themes, or webui.  If you choose example, then themes and webui are loaded automatically.
  • Click the Open Project Folder button
  • In the Projects pane, right click on each project and select the Build Project menu item.  The projects must be built in this order:  themes, webui, example.
You can deploy <myWorkspace>/woodstock/example/dist/example.war to any supported web container container using
the mechanisms available for that container, or thru Netbeans.
